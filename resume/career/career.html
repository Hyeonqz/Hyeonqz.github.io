배경 신규 가맹점 접수 시 한글명을 영문으로 직역으로 생성하여 가독성 저하 → 운영 담당자가 B/O에서 월 약 100건을 수동 수정하며, 담당자 부재 시 가맹접 접수 누락·지연 발생

결과RDB 직접 조회: 거래당 240→0회  (Redis 최우선 조회, 쓰기는 상태 변경 시에만) Pinpoint APM 기반 측정 결과 API 응답 200ms→30ms (85%↓)   [상세보기]

모놀리식 애플리케이션 → 멀티 모듈 및 DDD 기반 이벤트 주도 아키텍처(EDA) 전환

배경 단일 모놀리식에서 공통 코드가 비대화돼 거의 모든 변경이 공통 영역을 건드림 → merge 충돌·테스트 상호 간섭·배포-수정 반복으로 팀 병렬 작업이 막힘. 특정 기능의 장애가 전체 서비스로 전파되고, 모듈 간 동기 결합은 장애 전파·향후 MSA 분리 시 병목 리스크로 이어짐.
해결 1. Gradle 멀티 모듈 분리(서버 아키텍쳐 기준 API/TCP 경계 정렬) → 빌드·테스트·소유권 독립
2. 패키지 레벨 헥사고날 아키텍로 도메인을 인프라에서 격리(포트/어댑터) -> 기술 교체 유연성 확보
3. ApplicationEventPublisher + @TransactionalEventListener(AFTER_COMMIT)로 모듈 간 동기 트랜잭션 결합 제거
결과 모듈 경계 독립화: 빌드·테스트·소유권 분리로 팀 병렬 작업 가능 (부수적으로 Gradle 빌드 55→35s) [상세보기]
트랜잭션 결합 제거: 모듈 간 동기 호출을 AFTER_COMMIT 이벤트로 전환 → 한 모듈 트랜잭션이 타 모듈로 전파되지 않음
도메인-인프라 격리: 포트/어댑터로 기술 교체 유연성·테스트 용이성 확보

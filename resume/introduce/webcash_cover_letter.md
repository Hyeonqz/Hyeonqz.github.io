## 1. 지원 동기 및 성장 과정

저는 (주)큐뱅에서 핀테크 결제 도메인 서버 개발자로 근무하며, 알리페이·위챗페이·라인페이·유니온페이 등 국내외 간편결제사 7개사 이상과 REST API를 통해 직접 연동하는 시스템을 설계하고 운영해 왔습니다. 외부 금융 기관의 API 스펙을 분석하고, 연동 인터페이스 15개 이상을 직접 정의하며, 이기종 시스템 간 데이터 정합성을 유지하는 것이 얼마나 중요한지 몸소 경험했습니다.

쿠콘은 국내 최대 규모의 금융 데이터 API 연계 플랫폼으로서, 은행·카드·보험 등 수백 개의 금융기관 데이터를 기업 고객에게 안정적으로 제공하는 핵심 핀테크 인프라를 운영하고 있습니다. 제가 실무에서 쌓아온 다기관 API 연동 설계, 분산 환경에서의 데이터 정합성 확보, 그리고 금융 시스템의 안정적 운영 경험은 쿠콘의 API 플랫폼 고도화에 즉시 기여할 수 있는 역량이라고 확신합니다. 금융 데이터 허브로서 더 많은 기관과 고객을 연결하는 쿠콘의 비전에 함께하고 싶어 지원하게 되었습니다.

## 2. 직무 역량 및 핵심 경험

웹 캐시 그룹 중에 쿠콘의 핵심 사업인 **금융 데이터 API 연계**와 가장 직접적으로 연결된 경험을 중심으로 말씀드리겠습니다.

첫째, **다기관 외부 API 연동 설계** 경험입니다. 각 간편결제사마다 다른 인증 방식·응답 포맷·재시도 정책을 표준화하여 내부 연동 레이어를 설계했습니다. 이 구조 덕분에 신규 결제사 추가 시 개발 공수를 최소화하고, 일 평균 거래량 50건에서 4,000건(80배)으로 성장하는 과정에서도 무장애 운영을 달성했습니다.

둘째, **멱등성·정합성 기반 API 설계** 경험입니다. Final API 패턴을 도입하여 외부 기관과의 상태 불일치 문제를 구조적으로 해결하고, CS 문의를 월 10건에서 0~1건(95%↓)으로 낮췄습니다. 금융 데이터를 수백 개 기관과 연계하는 쿠콘 환경에서 필수적인 역량입니다.

셋째, **이기종 레거시 시스템 통합** 경험입니다. C++ DLL 기반 구형 POS와 Java 서버를 연동하는 네이티브 모듈을 직접 개발하여 이기종 시스템 통합을 실현했습니다. 쿠콘이 연동해야 하는 다양한 금융기관의 레거시 시스템 환경과 직결되는 경험입니다.

넷째, **고가용성 비동기 처리 아키텍처** 경험입니다. Kafka CorrelationId 기반 Request-Reply 설계로 분산 환경 타임아웃을 0%로 해소했고, Outbox 패턴으로 SFTP 파일 전송 유실률을 0%로 낮춰 10,000건 무장애 처리를 달성했습니다.

## 3. 입사 후 포부

웹 캐시 그룹에 입사하면, 금융 데이터 API 연계의 신뢰성과 확장성을 높이는 데 집중하겠습니다. <br>
단기적으로는 쿠콘이 연동하는 금융기관별 API 스펙과 데이터 흐름을 빠르게 파악하고, 현재 연동 구조에서 장애 위험·병목·정합성 취약점을 직접 발굴해 개선 제안을 할 수 있습니다.<br>
스타트업에서 쌓은 빠른 도메인 학습과 실행력으로 즉각적인 기여를 할 수 있습니다. <br>
중장기적으로는 마이데이터·오픈뱅킹 환경에서 점점 다양해지는 연동 요구사항을 수용할 수 있도록, 멱등성 보장·이벤트 드리븐 아키텍처·캐싱 전략을 결합한 확장 가능한 API 게이트웨이 구조 설계에 기여하고 싶습니다. <br>
금융 데이터 연결의 품질이 곧 쿠콘 고객사의 서비스 품질이라는 책임감을 가지고, 국내 금융 데이터 허브의 기술 기준을 높이는 개발자가 되겠습니다.
